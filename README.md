# OpenDART Financial Analyzer

OpenDART API를 이용하여 기업 정보, 공시 및 재무제표 데이터를 수집하고 SQLite에 저장한 뒤, 재무지표와 위험 공시를 분석하는 Python 기반 금융 데이터 분석 프로젝트입니다.

현재는 OpenDART의 기업 고유번호 목록을 내려받아 데이터베이스와 동기화하는 기능까지 구현되어 있습니다.

---

## 1. 프로젝트 목적

전자공시시스템 DART에는 기업의 사업보고서, 재무제표, 주요사항보고서 등 다양한 공시 데이터가 축적되어 있습니다.

하지만 OpenDART API를 활용하려면 다음과 같은 전처리 과정이 필요합니다.

* 기업별 DART 고유번호 관리
* 종목코드와 DART 고유번호 연결
* API 응답 검증 및 예외 처리
* XML 및 ZIP 데이터 파싱
* 수집 데이터의 로컬 저장
* 기업 정보 변경 및 삭제 이력 관리
* 재무제표 계정과목 정규화
* 기업별·연도별 재무비율 계산
* 위험 공시 탐지 및 분석

이 프로젝트는 이러한 과정을 하나의 데이터 수집·저장·분석 파이프라인으로 구성하는 것을 목표로 합니다.

## 3. 현재 구현 기능

### OpenDART API 공통 클라이언트

* 인증키 자동 포함
* 공통 GET 요청 처리
* JSON 응답 처리
* ZIP 파일 등 바이너리 응답 처리
* HTTP 오류 처리
* OpenDART 상태 코드 검증
* 빈 응답 검증
* 사용자 정의 `DartAPIError` 예외 제공

### 기업 고유번호 동기화

OpenDART의 `/corpCode.xml` API를 사용하여 전체 기업 목록을 동기화합니다.

처리 과정은 다음과 같습니다.

1. OpenDART에서 기업 고유번호 ZIP 파일 다운로드
2. ZIP 파일 내부의 XML 파일 추출
3. XML에서 기업 정보 파싱
4. SQLite 데이터베이스에 신규 저장 또는 갱신
5. 최신 목록에서 사라진 기존 기업 비활성화
6. 동기화 결과 요약 출력

수집하는 기업 정보는 다음과 같습니다.

* DART 기업 고유번호
* 기업명
* 주식 종목코드
* DART 정보 수정일
* 활성 상태
* 최초 확인 시각
* 최근 확인 시각
* 비활성화 시각

### SQLite 데이터베이스

기업 정보는 `data/dart.db` 파일에 저장됩니다.

프로그램 최초 실행 시 `data` 디렉터리와 필요한 테이블이 자동으로 생성됩니다.

현재 구현된 테이블:

#### `dart_corporations`

| 컬럼               | 설명             |
| ---------------- | -------------- |
| `corp_code`      | DART 기업 고유번호   |
| `corp_name`      | 기업명            |
| `stock_code`     | 주식 종목코드        |
| `modify_date`    | DART 기업 정보 수정일 |
| `is_active`      | 현재 기업 목록 포함 여부 |
| `first_seen_at`  | 최초 확인 시각       |
| `last_seen_at`   | 최근 확인 시각       |
| `deactivated_at` | 비활성화 시각        |

### 기업 정보 저장 및 조회

현재 Repository 계층에서 다음 기능을 제공합니다.

* 단일 기업 저장 또는 갱신
* 여러 기업 일괄 저장 또는 갱신
* DART 기업 고유번호 기준 조회
* 주식 종목코드 기준 조회
* 전체 기업 목록 조회
* 활성 기업만 조회
* 최신 DART 목록에서 누락된 기업 비활성화

기업 정보 갱신에는 SQLite의 UPSERT 방식을 사용합니다.

기존 기업이 다시 수집되면 다음 정보가 갱신됩니다.

* 기업명
* 종목코드
* 수정일
* 최근 확인 시각
* 활성 상태

기존 데이터는 물리적으로 삭제하지 않고 `is_active` 값을 변경하여 이력을 보존합니다.

### 동기화 결과 출력

프로그램 실행 후 다음 정보가 출력됩니다.

* OpenDART에서 수신한 전체 기업 수
* 저장 또는 갱신한 기업 수
* 종목코드를 보유한 기업 수
* 종목코드가 없는 기업 수
* 비활성화된 기업 수
* 동기화 시각

## 8. 환경변수 설정

프로젝트 최상위 디렉터리에 `.env` 파일을 생성합니다.

```env
crtfc_key=YOUR_OPENDART_API_KEY
```

OpenDART 인증키는 금융감독원 OpenDART 사이트에서 발급받을 수 있습니다.

보안을 위해 실제 `.env` 파일은 Git 저장소에 업로드하지 않습니다.

`.env.example`에는 실제 인증키가 아닌 변수 형식만 작성합니다.

```env
crtfc_key=
```

## 10. 데이터 동기화 정책

기업 데이터는 단순히 삭제하고 다시 저장하지 않고 동기화 방식으로 관리합니다.

### 신규 기업

데이터베이스에 존재하지 않는 `corp_code`가 수집되면 새로운 기업으로 저장합니다.

### 기존 기업

이미 존재하는 `corp_code`가 다시 수집되면 다음 정보를 갱신합니다.

* 기업명
* 종목코드
* 수정일
* 최근 확인 시각
* 활성 상태

`first_seen_at`은 최초 저장 시각을 유지합니다.

### 누락 기업

기존 데이터베이스에는 존재하지만 최신 OpenDART 기업 목록에 포함되지 않은 기업은 삭제하지 않습니다.

대신 다음과 같이 처리합니다.

```text
is_active = 0
deactivated_at = 동기화 시각
```

이를 통해 기업의 삭제, 합병, 청산 또는 DART 목록 변경 가능성을 추적할 수 있습니다.

### 재등장 기업

비활성화되었던 기업이 이후 목록에서 다시 확인되면 다음과 같이 복구됩니다.

```text
is_active = 1
deactivated_at = NULL
```

## 11. 예외 처리

현재 다음 상황에 대한 예외 처리가 구현되어 있습니다.

* OpenDART HTTP 요청 실패
* OpenDART 오류 상태 코드 반환
* 빈 API 응답
* 올바르지 않은 ZIP 파일
* ZIP 파일 내부에 XML 파일이 없는 경우
* XML 파싱 실패
* XML에서 유효한 기업 정보를 찾지 못한 경우
* 필수 기업 고유번호 누락
* 필수 기업명 누락
* 예상하지 못한 프로그램 오류

OpenDART API 관련 오류는 `DartAPIError`, 기업 동기화 관련 오류는 `CorporationSyncError`로 구분합니다.

프로젝트가 확장되면 다음 테이블을 추가할 예정입니다.

```text
dart_corporations
    └── 기업 기본정보 및 DART 고유번호

dart_disclosures
    └── 기업별 공시 목록과 공시 메타데이터

dart_financial_statements
    └── 기업별·연도별·보고서별 재무제표

dart_financial_accounts
    └── 정규화된 재무제표 계정과목

financial_ratios
    └── 계산된 주요 재무비율

disclosure_risk_flags
    └── 위험 공시 탐지 결과
```

기업 정보, 공시, 재무제표와 분석 결과를 분리하여 원본 데이터와 가공 데이터를 함께 관리할 예정입니다.

---

## 14. 설계 원칙

### 계층 분리

API 요청, 비즈니스 로직, 데이터베이스 접근을 서로 다른 모듈로 분리합니다.

```text
Client → Service → Repository → Database
```

이를 통해 각 계층의 책임을 명확하게 하고 테스트 및 유지보수를 쉽게 합니다.

### 원본 데이터 보존

OpenDART에서 수집한 데이터는 가능한 한 원본 형태로 저장하고, 분석 결과는 별도 테이블에서 관리할 예정입니다.

### 이력 관리

기업 목록에서 사라진 기업을 삭제하지 않고 비활성화하여 변경 이력을 추적합니다.

### 중복 방지

기업 고유번호, 접수번호, 사업연도, 보고서 코드 등의 고유 식별자를 이용해 중복 저장을 방지할 예정입니다.

### 재실행 가능성

동일한 수집 작업을 반복 실행해도 데이터가 불필요하게 중복되지 않도록 UPSERT 방식으로 설계합니다.

### 분석과 수집의 분리

데이터 수집 기능과 재무 분석 기능을 분리하여, 수집된 데이터를 여러 분석 로직에서 재사용할 수 있도록 구성합니다.

## 16. 주의사항

* 본 프로젝트는 학습 및 포트폴리오 목적으로 개발되고 있습니다.
* 분석 결과는 투자 판단을 위한 참고자료일 뿐 투자 권유가 아닙니다.
* OpenDART API 이용 정책과 호출 제한을 준수해야 합니다.
* 실제 API 인증키는 Git 저장소에 업로드하지 않아야 합니다.
* 기업 목록에서 누락된 기업이 반드시 폐업 또는 상장폐지를 의미하는 것은 아닙니다.
* 공시 및 재무제표 데이터는 제출 기업의 보고 내용과 OpenDART 제공 범위에 따라 달라질 수 있습니다.

## 17. 현재 개발 상태

현재 프로젝트는 **기업 기준정보 구축 단계**입니다.

OpenDART의 전체 기업 고유번호 목록을 내려받아 XML을 파싱하고, 기업명과 종목코드를 SQLite 데이터베이스에 동기화할 수 있습니다.

다음 개발 단계에서는 기업별 공시 목록을 수집하고 저장하는 기능을 구현할 예정입니다.
